1. Introduction ¶

1.1 Phase 1 Summary ¶

In Phase 1 of our project, we explored the Bank Marketing Dataset from Kaggle, which contains client demographic, financial and marketing campaign data collected during Portuguese bank telemarketing campaigns. The dataset includes 11,162 records and 17 initial features, with the target variable being whether a client subscribed to a term deposit.

To streamline analysis and reduce computational load, we randomly sampled 5,000 records using a fixed seed for reproducibility. We conducted comprehensive data cleaning and preprocessing, starting with the removal of irrelevant or low-quality features (contact, day, pdays) that were either redundant, sparsely populated or had limited predictive value. We validated data types, confirmed there were no missing values, and ensured categorical values were consistently formatted. We also consolidated infrequent or ambiguous categories. For example, merging "other" into "failure" in the poutcome feature to improve interpretability.

We identified and retained outliers in variables like balance and previous, recognising that extreme financial values could reflect important customer behaviours. Our final cleaned dataset consisted of 15 predictor features and one binary target variable deposit.

In the Data Exploration & Visualisation section, we used univariate, bivariate and multivariate visualisations to explore feature distributions and relationships with the target variable. Key findings included:

  • Higher subscription rates among clients with tertiary education, single marital status, and no housing or personal loans.

  • A strong positive correlation between call duration and subscription likelihood.

  • The outcome of previous campaigns and job type were also significant indicators of client responsiveness.

These insights inform our feature selection and modelling strategies in Phase 2. We aim to build robust classifiers by leveraging the most influential features identified in Phase 1 and evaluate multiple machine learning models to determine the best approach for predicting term deposit subscriptions.

1.3 Overview of Methodology ¶

Our Phase 2 modelling methodology follows a systematic machine learning pipeline designed to ensure rigorous evaluation, reliable performance and practical interpretability for marketing decision-making.

Data Preparation:¶

We begin by importing the cleaned dataset of 5,000 records developed in Phase 1. The target variable deposit is encoded into binary format. Categorical features are transformed via one-hot encoding, and numerical features are standardised using StandardScaler. The data is split using a 80:20 stratified train-test split to maintain class proportions and ensure reliable model evaluation.

Feature Selection:¶

We apply SelectKBest with ANOVA F-statistics (f_classif) to identify the most discriminative features relative to the binary target. The top 10 features are retained for modelling, improving computational efficiency and reducing the risk of overfitting. Feature importance scores are visualised to support model transparency and interpretation.

Model Fitting & Hyperparameter Tuning:¶

We implement six classification algorithms from different Scikit-Learn submodules:

  • Logistic Regression

  • K-Nearest Neighbours (KNN)

  • Decision Tree

  • Random Forest

  • Support Vector Machine (SVM)

  • Neural Network (Advanced)

Each model is wrapped in a Pipeline to integrate scaling, feature selection and classifier tuning. We conduct hyperparameter tuning using GridSearchCV with 5-fold stratified cross-validation, optimising primarily for AUC (Area Under the ROC Curve). The rationale for using AUC is to address the mild class imbalance in the dataset; AUC provides a robust evaluation metric by incorporating sensitivity and specificity across all classification thresholds.

For each model, we document:

  • Best parameter sets

  • Performance metrics on the test set (accuracy, recall, F1-score)

  • Hyperparameter tuning visualisations (line plots and heatmaps)

Neural Network Model Fitting & Tuning:¶

A neural network model is built using TensorFlow/Keras, starting with a baseline architecture and refined through iterative tuning of:

  • Neuron count

  • Activation function

  • Optimizer

  • Dropout rate

  • Batch size

Each tuning step includes accuracy tracking and visual interpretation. The final neural network uses a two-layer structure, ReLU activations, Adam optimiser, and achieves strong accuracy and AUC comparable to ensemble methods.

Model Comparison:¶

All models are evaluated using 5-fold cross-validation, and the best models are compared using accuracy and AUC. Paired t-tests are performed to assess the statistical significance of performance differences. The results are presented in tabular form and visualised using boxplots and bar charts for clarity.

# Encode the target variable
df['deposit'] = df['deposit'].map({'no': 0, 'yes': 1})

X = df.drop('deposit', axis=1)

selector = SelectKBest(score_func=f_classif, k=10)  # select top 10 features
X_new = selector.fit_transform(X_encoded, y)

scores = selector.scores_

plt.ylabel('Feature')
plt.tight_layout()
plt.show()

Feature Selection Using ANOVA F-score

We perform a univariate analysis using the ANOVA F-statistic (f_classif) to identify the main predictors of client subscription to a term deposit. In classification problems where the target variable is binary (as in "deposit"), Logistic Regression is typically the preferred approach.

Categorical features are encoded as dummy variables using the pd.get_dummies() function with the drop_first=True parameter to prevent multicollinearity. The transformed feature set is then passed to SelectKBest(score_func=f_classif, k=10) to identify the ten features most strongly associated with the target variable based on F-statistics.

Key Findings from the Feature Scores

The resulting F-score rankings are visualised in the figure above. These scores indicate the discriminatory power of each feature, quantifying the variance between the “yes” and “no” classes for a given variable.

The most informative features identified are:

  • duration: By far, having a long time selected is the most important predictor of people opening new term deposits. Experience also tells us that more conversations with a client often reflect greater desire or persuasion.

  • poutcome_success: Clients who previously responded positively to a marketing campaign are far more likely to subscribe again, confirming the predictive power of historical campaign performance.

  • poutcome_unknown: The uncertainty around previous campaign outcomes also carries predictive weight, perhaps signaling segmentation or data gaps in client history.

  • Duration-based groups (duration_group_601_800, duration_group_801_1000, etc.): These engineered features, derived from groupings of the continuous duration variable, also score highly, reaffirming the dominant influence of call duration.

  • housing_yes: Suggests some correlation between homeownership and deposit subscription, potentially due to greater financial maturity or stability.

  • month_may: Temporal patterns in campaign effectiveness are reflected here, as May appears to be a significant month, possibly due to internal bank strategies or external socio-economic conditions.

  • age_group_61-70: This demographic segment also exhibits higher relevance, which may be related to nearing retirement or changes in investment behaviour.

Justification for Use in Predictive Modelling

Reducing the feature space to the most informative variables offers several advantages:

  • Improved generalisation by reducing noise and the risk of overfitting.

  • Faster and more efficient training due to lower computational requirements.

  • Enhanced interpretability by focusing on the key factors influencing client decisions.

These chosen features will be used to train and set up different classification algorithms in the next part of the process. Because of this, our models are statistically reliable and understandable by others, which is recommended in practical machine learning.

2.2.1 Logistic Regression ¶

In [4]:
from sklearn.linear_model import LogisticRegression

# Pipeline with scaling
pipeline_lr = Pipeline([
    ('scaler', StandardScaler()),
    ('clf', LogisticRegression(solver='liblinear'))
])

# Hyper-parameter grid
param_grid_lr = {
    'clf__C': [0.01, 0.1, 1, 10],
    'clf__penalty': ['l1', 'l2']
}

# Grid search
grid_lr = GridSearchCV(pipeline_lr, param_grid_lr, cv=5, scoring='accuracy')
grid_lr.fit(X_train, y_train)

# Results
print("Best Logistic Regression Params:", grid_lr.best_params_)
print("Test Accuracy:", accuracy_score(y_test, grid_lr.predict(X_test)))
print(classification_report(y_test, grid_lr.predict(X_test)))

# Plot
results_lr = pd.DataFrame(grid_lr.cv_results_)

Performance is evaluated by tuning the regularisation parameter (C) and selecting the type of regularisation (L1 or L2). A moderate C value combined with an L2 penalty generally yields the best results, helping to prevent overfitting while maintaining predictive accuracy. L2 regularisation typically produces small, but not exactly zero coefficients, making it suitable for models with many features.

Mixed feature sets result in stable accuracy for both L1 and L2 regularisation, but the best performance consistently comes from L2. The sensitivity of L1 regularisation to the choice of C is likely due to its tendency to eliminate less significant features, which can either enhance or degrade performance depending on the dataset.

Model Performance and Insights

The best-performing model (C=0.1, penalty='l2') achieves a test accuracy of 0.791, the highest among the evaluated models:

  • Class 1: Precision = 0.81, Recall = 0.73, F1-score = 0.77

  • Macro and weighted F1-scores: 0.79

Slightly better recall is observed for the no deposit class, while precision is higher for the deposit class. Logistic Regression provides a good representation of the main trends in the data using a relatively simple model structure.

For binary classification tasks, Logistic Regression offers simple and quick performance and helps us easily identify the main features by examining its coefficients. Although its reliance on linear decision boundaries may limit effectiveness with more complex data, but adding regularisation (mainly L2) improves its generalisability. Its straightforward structure makes it a reliable baseline model, and its results support that the process of selecting features is solid. Future improvements could include incorporating interaction terms, generating polynomial features, or exploring nonlinear methods such as kernel-based models.

Evaluation Metrics and Their Significance

Accuracy indicates the percentage of correct predictions out of all predictions. While it is straightforward and easy to understand, it does not account well for performance nuances in imbalanced datasets.

AUC is an effective metric for binary classification because it considers the relationship between the true positive rate and the false positive rate as thresholds vary. A higher AUC indicates better model separation between positive and negative classes.

This highlights that while Random Forest slightly outperforms other models in mean accuracy and AUC, its improvement is not statistically robust over Logistic Regression and SVM but is significant over KNN.

Critical Reflection

While Random Forest emerges as the top-performing model based on both metrics and statistical testing, it is important to consider model complexity and interpretability. Logistic Regression, though slightly less accurate, offers greater interpretability and ease of deployment, making it a practical choice for regulated environments such as banking.

In contrast, the Neural Network’s unstable performance suggests that either:

  • The network is too complex for the dataset size,

  • Further hyperparameter tuning (e.g., early stopping patience, learning rate) is needed, or

  • More data is required to fully realise the potential of deep learning in this context.

Additionally, paired t-tests assume normality of differences and may lack robustness with a very small sample size (n = 5 folds). Therefore, conclusions from these tests should be interpreted with caution.

Conclusion

Overall, the Random Forest model demonstrates the strongest and most consistent performance, closely followed by Logistic Regression and SVM. The Neural Network, despite promising AUC scores, suffers from high variance and lower accuracy, limiting its reliability. KNN is consistently outperformed and can be excluded from further consideration. These findings inform the selection of a final model by balancing predictive performance, interpretability and deployment feasibility.

3. Critique & Limitations ¶

Our approach follows a structured machine learning process that includes data cleaning, feature selection, model training and evaluation. One of the main strengths is the use of multiple algorithms, including Logistic Regression, Decision Tree, Random Forest, KNN, SVM and Neural Network. This allows us to compare different modelling techniques and choose the most suitable one. We also apply 5-fold cross-validation and use AUC as the main evaluation metric, which is appropriate given the imbalance in the target variable.

We use pipelines and grid search to tune hyperparameters, helping us improve model performance in a consistent and reliable way. The use of SelectKBest for feature selection also simplifies the model and reduces the risk of overfitting. In the advanced section, we build and tune a neural network by adjusting several hyperparameters, supported by visual plots and performance summaries.

However, there are some limitations. We only use 5,000 records out of over 11,000 available, which may limit the learning ability of more complex models like neural networks. While we consider class imbalance in our evaluation using AUC, we do not apply resampling techniques such as SMOTE during training. This may affect how well the models handle minority classes.

The neural network shows higher AUC but lower accuracy and more variability across folds. This suggests it may not generalise as well on this relatively small dataset. Also, we do not perform much feature engineering or bring in domain knowledge, which could help improve predictions.

Overall, our approach demonstrated a solid foundation in practical machine learning with a strong balance between performance, transparency and extensibility. However, future improvements could focus on enhancing class imbalance handling, integrating domain knowledge, and testing models on unseen real-world data for deployment readiness.

4. Summary & Conclusions ¶

4.1 Project Summary ¶

In this project, we aim to predict whether a bank client will subscribe to a term deposit based on their personal, financial and campaign-related information. In Phase 1, we begin by exploring the Bank Marketing Dataset, which contains 11,162 records. To reduce computation time, we take a random sample of 5,000 observations for our analysis. We then perform data cleaning by removing irrelevant features such as contact, day and pdays, and ensure all data types are correctly formatted. We check for missing values and find none. We clean categorical variables and merge redundant categories like "other" in the poutcome column. We also examine outliers but choose to retain them, as they may reflect valid customer behaviour.

Next, we perform exploratory data analysis (EDA) using univariate, bivariate and multivariate visualisations. We find that features such as call duration, previous campaign outcome, education level, marital status and loan status have strong relationships with the target variable. These insights guide our feature selection in the next phase.

In Phase 2, we prepare the cleaned dataset for modelling. We apply one-hot encoding to categorical variables and standardise numerical features using StandardScaler. We split the data into 80% training and 20% testing sets using stratified sampling to maintain class distribution. We use SelectKBest with ANOVA F-statistics (f_classif) to select the top 10 most important features for prediction.

We then build and tune six machine learning models: Logistic Regression, K-Nearest Neighbours, Decision Tree, Random Forest, Support Vector Machine and Neural Network. Each model is trained using a pipeline with GridSearchCV and 5-fold stratified cross-validation. We evaluate performance using accuracy, F1-score, recall, and AUC, with AUC chosen as the main scoring metric due to the class imbalance in the dataset.

As part of the Advanced Submission, we develop a Neural Network model using TensorFlow/Keras. We tune five key hyperparameters, including neuron count, activation function, optimiser, dropout rate, and batch size. For each tuning step, we generate plots and discuss the impact on model performance. The final model achieves strong AUC and competitive accuracy.

Finally, we compare all models using cross-validated accuracy and AUC scores and perform paired t-tests to assess whether the differences between models are statistically significant. This comprehensive process allows us to identify the most effective and reliable model for predicting term deposit subscriptions.

4.2 Summary Findings ¶

Our analysis shows several important findings about the factors that influence a client’s decision to subscribe to a term deposit, as well as the performance of different machine learning models in predicting this outcome.

From the feature selection process, we find that call duration is by far the most influential predictor. Clients who spend more time on the phone are significantly more likely to subscribe. Other important features include the outcome of previous campaigns (poutcome_success), whether the client owns a house (housing_yes), the month of contact (month_may), and certain age and duration groupings. These findings confirm that both behavioural and temporal factors strongly affect subscription likelihood.

Among the machine learning models, Random Forest achieves the highest average accuracy (0.792) and AUC (0.872), followed closely by Logistic Regression and Support Vector Machine, both of which also perform consistently well across metrics. K-Nearest Neighbours performs the weakest among the models evaluated, both in terms of accuracy and AUC. The Neural Network model performs competitively in terms of AUC (0.855), suggesting it can effectively rank probability scores, but shows greater variability and slightly lower accuracy, likely due to the small dataset size.

Our paired t-tests reveal that the performance difference between Random Forest and Logistic Regression is not statistically significant, indicating that both models perform similarly in a practical sense. However, both models significantly outperform KNN, confirming its lower predictive capability for this dataset.

Overall, we find that simple and interpretable models like Logistic Regression can perform nearly as well as more complex ensemble methods. Neural Networks show potential but require further tuning and possibly a larger dataset to outperform classical methods. These results highlight that using call duration, past campaign success, and customer profile features can significantly improve the targeting of marketing campaigns.

4.3 Conclusions ¶

The goal of this project is to help the bank predict which clients are likely to subscribe to a term deposit, using data from past marketing campaigns. We achieve this by cleaning the data, exploring key patterns, selecting important features, and building machine learning models to make accurate predictions.

We find that call duration, previous campaign outcomes, and client characteristics such as loan status and education level are strong indicators of whether a client will subscribe. These insights can help the bank better target its marketing efforts by focusing on clients who are more likely to say yes.

Among the models we tested, Random Forest performs the best, with the highest accuracy and AUC score. Logistic Regression and SVM also perform well and are more interpretable. The Neural Network shows potential but is more sensitive to changes and performs less consistently on our dataset.

Using AUC as the main evaluation metric helps us account for the imbalance in the data, where most clients do not subscribe. This gives us a better measure of how well each model works.

In conclusion, our models can support smarter marketing decisions. By focusing on the right clients, the bank can improve campaign success and reduce wasted effort. For future work, we suggest using more data, adding new features and testing the models on live campaign data.
