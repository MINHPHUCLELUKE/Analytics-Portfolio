# Software & System Deconstruction using UML Models

Structural analysis of a real software system (KIS Booking) using UML modelling — reverse-engineering the system's architecture, behaviour, and interactions into formal diagrams.

---

## Approach

- Use case diagrams: mapping actors and their interactions with the system
- Sequence diagrams: tracing message flows and operation order across components
- Class/object diagrams: capturing data structures and relationships
- Activity diagrams: modelling process flows and decision points
- Written interpretation of each diagram connecting model structure to real system behaviour

---

## Purpose

UML deconstruction is a core analytical skill in systems thinking and software design. This project demonstrates the ability to read an existing system critically, identify its components, and communicate architecture precisely through standardised notation.
